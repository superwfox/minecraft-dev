# 使用方法

## 进入对话页

从首页点击「立即体验」或「开始对话」按钮，跳转到对话页面。

## 输入需求

在底部输入框中描述你的开发需求，按回车发送。例如：

```
用 Paper 1.20 写一个玩家加入时发送欢迎消息的插件
```

尽量在描述中包含核心类型（Paper/Bukkit/Spigot/Forge/Fabric）和游戏版本，这样可以跳过选择步骤，直接生成结果。

## 补充信息

如果输入中没有明确核心类型或版本，系统会弹出选择面板：

- 核心类型：PAPER / BUKKIT / SPIGOT / FORGE / FABRIC
- 版本：1.8 ~ 1.21

点击选择后按「确认」继续。

## 查看结果

系统返回结构化的开发步骤，每一步包含：

- 步骤描述（白色文本）
- 事件名称（蓝色标签）— 需要监听的事件，如 `PlayerJoinEvent`
- 方法名称（橙色标签）— 需要实现的方法
- 参数列表（灰色标签）— 方法的参数签名

顶部会显示当前使用的核心类型和版本。

## 继续对话

结果渲染完成后，底部会出现新的输入框，可以继续提问。后续对话默认使用流式输出，实时显示 AI 的回复文本。

## 导航栏状态

对话过程中，顶部导航栏的中间文字会实时反映当前阶段：

- 「正在分析需求...」— 第一阶段，提取参数
- 「正在生成开发步骤...」— 第二阶段，生成步骤
- 「渲染完成」— 结果已展示
- 「请求失败」— 出现错误，错误信息会显示在对话块中
