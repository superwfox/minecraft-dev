# 使用方法

## 进入对话页

从首页点击「立即体验」或「开始对话」按钮，跳转到对话页面。

## 输入需求

在底部输入框中描述你的开发需求，按回车发送。例如：

```
用 Paper 1.21 写一个玩家加入时发送欢迎消息的插件
```

尽量在描述中包含核心类型（Paper/Bukkit/Spigot/Forge/Fabric）和游戏版本，这样可以跳过选择步骤，直接生成结果。

## 补充信息

如果输入中没有明确核心类型或版本，系统会弹出选择面板：

- **核心类型**：PAPER / BUKKIT / SPIGOT / FORGE / FABRIC
- **版本**：1.8 ~ 1.21

选择后按「确认」继续。

## 查看开发步骤

系统返回结构化的开发步骤，每一步包含：

- 步骤描述（白色文本）
- 事件名称（蓝色标签）— 需要监听的事件
- 方法名称（橙色标签）— 需要实现的方法
- 参数列表（灰色标签）— 方法的参数签名

顶部会显示当前使用的核心类型和版本。

## 生成项目

步骤渲染完成后，底部出现「生成项目 & 构建 JAR」按钮。点击后系统自动：

1. 调用 Planner 规划项目文件结构
2. 逐个生成源文件（每个文件生成后自动审查）
3. 校验文件完整性
4. 上传到 GitHub 并触发 Maven 构建
5. 等待编译完成后显示下载按钮

整个过程分为四张卡片展示（规划 / 代码生成 / 构建 / 日志），导航栏文字实时反映当前阶段。

## 下载 JAR

构建成功后点击「📦 下载 JAR」，获得包含编译产物的压缩包。

## 导航栏状态

对话和生成过程中，顶部导航栏中间文字实时反映当前阶段：

| 阶段 | 导航栏文字 |
|------|-----------|
| 需求分析 | 正在分析需求... |
| 步骤生成 | 正在生成开发步骤... |
| 步骤完成 | 渲染完成 |
| 项目规划 | 正在规划项目... |
| 代码生成 | 正在生成代码... |
| 文件校验 | 正在校验文件... |
| 上传构建 | 正在上传构建... |
| 编译打包 | 正在编译打包... |
| 就绪 | JAR 已就绪 |
| 失败 | 生成失败 |
